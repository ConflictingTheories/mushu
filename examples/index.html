  // Compute shader (or null for render-only)

  }`

WebGL2 â€¢ Fluent Builder 
import { mushu, shader, fps } from 'mushu';

// Chainable, pluggable, beautiful
mushu('#canvas').flow()
  .use(shader(`
    void mainImage(out vec4 O, vec2 C) {
      vec2 uv = C / resolution;
      O = vec4(uv, 0.5 + 0.5 * sin(time), 1.0);
    }
  `))
  .use(fps())  // Add FPS counter plugin
  .go();      // Start the render loop

// WebGPU with fluent API

    @compute fn main(@builtin(global_invocation_id) id: vec3u) {
      // Physics simulation
    }
  `)
  .display(`
    @fragment fn main(@location(0) uv: vec2f) -> @location(0) vec4f {
      return vec4f(uv, sin(time), 1.0);
    }
  `)
  .go();

â“ Frequently Asked Questions

What's the difference between glsl() and flow()?

mushu().glsl() is a one-liner for quick shader sketches. mushu().flow() is a fluent builder that supports plugins like FPS counters, simulations, and post-processing chains.

Do I need WebGPU support?

No! The mushu().glsl() and mushu().flow() APIs use WebGL2, which works everywhere. WebGPU (mushu().gpu() and mushu().gpu().flow()) is optional for when you need compute shaders.

What uniforms are available?

All shaders get time (seconds), resolution (canvas size), and mouse (0-1 normalized position) automatically. No setup needed.

Can I use this for Shadertoy-style effects?

Absolutely! The mainImage(out vec4 O, vec2 C) signature is Shadertoy-compatible. Just paste your shader code.

How do I add mouse interaction?

The mouse uniform is already there! It's a vec2 with values 0-1. Use it in your shader for interactive effects.

Is mushu good for learning shaders?

Yes! Compare the boilerplate examples (200+ lines) to the mushu().glsl() versions (10 lines) to see how much easier it is to focus on the actual shader code.
